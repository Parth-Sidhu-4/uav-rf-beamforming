# Code of Conduct

## Context

This is a **closed academic research project** with no open contribution model.
There are no community contributors, no maintainer team, and no public development
activity. Accordingly, this Code of Conduct is intentionally minimal — it governs
interactions in the limited public-facing surface area of this repository
(i.e., the issue tracker, if open).

---

## Expected Behaviour

Anyone interacting with this repository — whether opening an issue, sending an
email, or otherwise engaging with the Author — is expected to:

- Communicate respectfully and professionally at all times.
- Clearly state the purpose of their enquiry (academic reference, licensing
  question, attribution concern, etc.).
- Respect that this is a student academic project with no obligation to respond,
  provide support, or accept contributions.
- Credit the Author (Parth Sidhu) appropriately if referencing this work in any
  academic or professional context, as described in `CITATION.cff` and `LICENSE`.

---

## Unacceptable Behaviour

The following will result in immediate blocking and, where applicable, reporting
to the relevant platform:

- Harassment, abuse, or intimidatory language directed at the Author.
- Attempts to claim ownership of, or misrepresent the origin of, this work.
- Submitting pull requests after being explicitly told the project is closed to
  contributions (see `CONTRIBUTING.md`).
- Any contact attempting to extract confidential technical information beyond what
  is published in this repository.
- Legal threats made outside the formal channels described in `LICENSE`.

---

## Scope

This Code of Conduct applies to:

- GitHub Issues on this repository.
- Any direct communication initiated in connection with this repository
  (e.g., email to parthecstasy@gmail.com referencing this project).

It does not apply to the Author's personal conduct in unrelated contexts.

---

## Enforcement

This repository is maintained solely by **Parth Sidhu**. Violations of this
Code of Conduct may result in:

- Closure and locking of the relevant issue without response.
- Blocking of the offending account from the repository.
- Reporting to GitHub Trust & Safety where platform terms are violated.
